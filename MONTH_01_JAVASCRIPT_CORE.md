# Month 01 — JavaScript Core + DOM

## Цель месяца

За 30 дней научиться руками писать базовую frontend-логику на чистом JavaScript:

- находить элементы на странице;
- реагировать на действия пользователя;
- хранить состояние;
- обновлять интерфейс;
- работать с массивами и объектами;
- валидировать формы;
- сохранять данные в `localStorage`;
- делать простую архитектуру проекта.

Главная формула месяца:

```text
пользователь сделал действие → событие → меняем состояние → обновляем UI
```

---

## Правила месяца

### Codex можно

- создать папку урока;
- создать `index.html`;
- создать `style.css`;
- создать пустой `script.js` с TODO;
- оформить README;
- проверить мой код;
- объяснить ошибку;
- предложить рефакторинг после моей реализации;
- подготовить commit message.

### Codex нельзя

- писать основную JS-логику за меня;
- писать обработчики событий за меня;
- писать render-функции за меня;
- решать финальную задачу месяца;
- коммитить код, который я не могу объяснить.

### Я сам пишу

- `querySelector` / `querySelectorAll`;
- состояние через `let`;
- обработчики событий;
- функции обновления UI;
- условия;
- работу с массивами;
- работу с `localStorage`;
- основную бизнес-логику.

# Главные вопросы месяца

## 1. Что такое DOM?

DOM — это представление HTML-страницы в виде объектов, с которыми работает JavaScript.

## 2. Что делает `querySelector`?

Ищет первый элемент по CSS-селектору.

## 3. Что делает `querySelectorAll`?

Ищет все элементы по CSS-селектору.

## 4. Что такое событие?

Событие — действие пользователя или браузера: клик, ввод, отправка формы, загрузка.

## 5. Что делает `addEventListener`?

Назначает функцию, которая выполнится при событии.

## 6. Что такое состояние?

Данные, от которых зависит интерфейс.

## 7. Почему после изменения состояния нужно обновить UI?

Потому что переменная JS не меняет HTML автоматически.

## 8. Что такое render-функция?

Функция, которая обновляет интерфейс на основе данных.

## 9. Зачем нужен `localStorage`?

Чтобы сохранить данные в браузере после перезагрузки.

## 10. Зачем нужны `JSON.stringify` и `JSON.parse`?

Чтобы сохранять массивы/объекты как строки и восстанавливать их обратно.
