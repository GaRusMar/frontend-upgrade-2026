# Правила Codex

## Можно

- создать структуру
- создать файлы
- написать базовую вёрстку
- подготовить CSS
- написать README
- сделать commit message
- найти ошибку
- объяснить ошибку
- предложить рефакторинг
- сделать code review

## Нельзя

- писать за тебя основную логику
- решать учебную задачу целиком
- коммитить непонятный код
- принимать diff без чтения
- копировать решение без объяснения

## Перед каждым коммитом ты должен ответить

1. Что изменилось?
2. Почему так?
3. Где состояние?
4. Где бизнес-логика?
5. Как проверить?
6. Что может сломаться?
