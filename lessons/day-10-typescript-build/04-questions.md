## Ответы

1. Чтобы собрать модули, обработать TypeScript и подготовить код для браузера.

3. Главный файл, с которого начинается приложение.

5. Отдаёт код из файла.
6. Получает код из другого файла.
7. Чтобы находить ошибки типов до запуска кода.
8. Нет, TypeScript компилируется в JavaScript.
9. Явное указание типа.
10. `Product[]`.
11. Описание формы объекта.
12. Необязательное свойство через `?`.
13. Тип “одно из нескольких значений”.
14. Чтобы TypeScript ловил опечатки.
15. Interface часто для объектов, type часто для union/alias.
16. `function fn(price: number): string`.
17. Функция не возвращает полезное значение.
18. Он отключает пользу TypeScript.
19. Конфигурация TypeScript.
20. Включает строгую проверку типов.
21. Запрещает неявный any.
22. Команды в package.json.
23. Проверяет типы без запуска кода.
24. Собирает проект.
25. Он уменьшает runtime-ошибки и помогает работать в команде.
