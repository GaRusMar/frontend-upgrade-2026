# Day 01 — Теория

## 1. HTML, CSS и JavaScript

Во frontend у каждого слоя своя роль.

```text
HTML       → структура страницы
CSS        → внешний вид
JavaScript → поведение и логика
```

Например, в нашем калькуляторе:

```text
HTML создаёт поля и кнопки
CSS делает интерфейс красивым
JavaScript считает цену заказа
```

## 2. Подключение JavaScript

Обычно файл JavaScript подключается перед закрывающим тегом `body`.

```html
<script src="./script.js"></script>
</body>
```

Почему внизу?

Потому что браузер сначала должен прочитать HTML-элементы, а потом JavaScript сможет их найти.

Если подключить скрипт слишком рано, JavaScript может попытаться найти элемент, которого ещё нет на странице.

## 3. Комментарии

Комментарии помогают оставлять пояснения в коде.

Однострочный комментарий:

```js
// это комментарий
```

Многострочный комментарий:

```js
/*
  это многострочный
  комментарий
*/
```

В учебном проекте комментарии `TODO` показывают, что нужно сделать самостоятельно.

## 4. Переменные

Переменная — это имя для значения.

В современном JavaScript чаще всего используют:

```js
const
let
```

## 5. const

`const` используют, если переменную не нужно переопределять.

Например, мы нашли кнопку:

```js
const calculateButton = document.querySelector('.calculator__button')
```

Мы не собираемся записывать в `calculateButton` другую кнопку.

Поэтому используем `const`.

Важно:

```text
const запрещает переопределить переменную,
но не делает объект или DOM-элемент полностью неизменяемым
```

## 6. let

`let` используют, когда значение может меняться.

Например:

```js
let discount = 0
```

Если пользователь ввёл промокод, скидка изменится:

```js
discount = itemsTotal * 0.1
```

Значит, здесь подходит `let`.

## 7. Типы данных

В этом уроке нужны базовые типы:

```text
string  → строка
number  → число
boolean → true / false
```

Примеры:

```js
const promoCode = 'BANGKOK10'
const price = 16
const hasDiscount = true
```

## 8. input.value всегда строка

Даже если input имеет `type="number"`, значение приходит в JavaScript как строка.

```js
const priceValue = priceInput.value
```

Если пользователь ввёл `16`, в JavaScript это будет:

```js
'16'
```

Это строка, не число.

## 9. Почему это важно

Сравни:

```js
'16' + '2'
```

Результат:

```text
"162"
```

Потому что строки склеиваются.

А вот так:

```js
16 + 2
```

Результат:

```text
18
```

Поэтому перед расчётами строки из input нужно превращать в числа.

## 10. Number()

`Number()` преобразует значение в число.

```js
const price = Number(priceInput.value)
```

Примеры:

```js
Number('16')     // 16
Number('16.5')   // 16.5
Number('')       // 0
Number('hello')  // NaN
```

## 11. NaN

`NaN` означает `Not a Number`.

Это специальное значение появляется, когда JavaScript не смог получить нормальное число.

Например:

```js
Number('hello')
```

Результат:

```js
NaN
```

## 12. Number.isNaN()

Проверять `NaN` нужно через `Number.isNaN`.

```js
if (Number.isNaN(price)) {
  // показать ошибку
}
```

Нельзя надёжно проверять так:

```js
price === NaN
```

Это частая ошибка новичков.

## 13. Арифметика

В этом уроке нужны операции:

```text
+ сложение
- вычитание
* умножение
/ деление
```

Пример:

```js
const itemsTotal = dishPrice * quantity
```

## 14. Условия

Условие позволяет выполнить код только в определённой ситуации.

```js
if (itemsTotal >= 30) {
  delivery = 0
} else {
  delivery = deliveryPrice
}
```

Это читается так:

```text
если сумма товаров больше или равна 30,
то доставка бесплатная,
иначе доставка платная
```

## 15. Сравнение

В JavaScript важно различать:

```text
=   присваивание
=== строгое сравнение
```

Присваивание:

```js
discount = 10
```

Сравнение:

```js
promoCode === 'BANGKOK10'
```

Ошибка:

```js
if (promoCode = 'BANGKOK10') {
  // плохо: это присваивание, а не сравнение
}
```

Правильно:

```js
if (promoCode === 'BANGKOK10') {
  // хорошо
}
```

## 16. Boolean

Boolean — это логический тип.

Он имеет два значения:

```js
true
false
```

Пример:

```js
const hasPromoCode = promoCode === 'BANGKOK10'
```

Если промокод совпал, `hasPromoCode` будет `true`.

Если не совпал, будет `false`.

## 17. Получение элементов со страницы

Чтобы JavaScript мог работать с HTML, нужно найти элементы.

```js
const form = document.querySelector('.calculator__form')
```

`querySelector` ищет первый элемент по CSS-селектору.

Класс в селекторе пишется с точкой:

```js
'.calculator__form'
```

## 18. Событие submit

Форма отправляется через событие `submit`.

```js
form.addEventListener('submit', (event) => {
  // код
})
```

Слушать лучше форму, а не кнопку, потому что submit сработает и по кнопке, и по Enter.

## 19. preventDefault()

По умолчанию форма перезагружает страницу.

Чтобы этого не было:

```js
event.preventDefault()
```

Это значит:

```text
браузер, не отправляй форму сам,
я обработаю её через JavaScript
```

## 20. textContent

Чтобы вывести результат на страницу, используем `textContent`.

```js
totalElement.textContent = finalTotal
```

Для обычного текста это безопаснее и проще, чем `innerHTML`.

## 21. console.log

`console.log` выводит значение в консоль браузера.

```js
console.log(dishPrice)
console.log(itemsTotal)
```

Это помогает понять, что происходит в коде.

Но после отладки лишние `console.log` лучше удалить.

## 22. debugger

`debugger` ставит точку остановки в коде.

```js
debugger
```

Если DevTools открыт, браузер остановит выполнение на этой строке.

Это позволяет пошагово посмотреть:

```text
какие значения в переменных
какая ветка if выполняется
где появляется ошибка
```

## 23. DevTools Console

Во вкладке Console можно увидеть:

```text
- ошибки JavaScript
- console.log
- предупреждения браузера
```

Если код не работает, сначала смотри Console.

## 24. DevTools Sources

Во вкладке Sources можно:

```text
- открыть script.js
- поставить breakpoint
- выполнить код по шагам
- посмотреть переменные
```

Это основной инструмент отладки.

## 25. Как читать ошибку

Обычно ошибка содержит:

```text
тип ошибки
сообщение
файл
номер строки
```

Например:

```text
Uncaught TypeError: Cannot read properties of null
script.js:12
```

Это значит:

```text
на строке 12 мы пытаемся работать с элементом,
который не найден
```

Часто причина — неправильный CSS-селектор.

## 27. Частые ошибки

### Ошибка 1. Забыть точку в querySelector

Неправильно:

```js
document.querySelector('calculator__form')
```

Правильно:

```js
document.querySelector('.calculator__form')
```

### Ошибка 2. Считать строки вместо чисел

Неправильно:

```js
const total = priceInput.value + deliveryInput.value
```

Правильно:

```js
const total = Number(priceInput.value) + Number(deliveryInput.value)
```

### Ошибка 3. Проверять NaN через ===

Неправильно:

```js
price === NaN
```

Правильно:

```js
Number.isNaN(price)
```

### Ошибка 4. Перепутать = и ===

Неправильно:

```js
if (promoCode = 'BANGKOK10')
```

Правильно:

```js
if (promoCode === 'BANGKOK10')
```

### Ошибка 5. Забыть preventDefault

Форма перезагрузит страницу, и результат исчезнет.

### Ошибка 6. Не очищать старую ошибку

Пользователь исправит данные, но старая ошибка останется на экране.

## 28. Что важно запомнить

```text
input.value всегда строка
для расчётов нужен Number()
ошибки смотрим в Console
сложные места проверяем через debugger
интерфейс обновляем через textContent
```
