# 07 — Interview Questions

## DOM and Rendering

### 1. Что такое DOM?

DOM — объектная модель документа. Браузер превращает HTML в дерево объектов, с которым JavaScript может работать.

### 2. Чем `textContent` отличается от `innerHTML`?

`textContent` вставляет текст. `innerHTML` вставляет HTML и может быть опасен при пользовательских данных.

### 3. Зачем нужна render-функция?

Она превращает данные или state в интерфейс.

## Events

### 4. Что делает `addEventListener`?

Назначает обработчик события на элемент.

### 5. Что такое `event.target`?

Элемент, на котором событие реально произошло.

### 6. Что такое `event.currentTarget`?

Элемент, на котором висит текущий обработчик.

### 7. Что такое всплытие событий?

Событие поднимается от целевого элемента к родителям.

### 8. Что такое делегирование событий?

Один обработчик на родителе управляет событиями дочерних элементов.

## State

### 9. Что такое state?

Данные, которые описывают текущее состояние интерфейса.

### 10. Почему DOM должен зависеть от state?

Так интерфейс становится предсказуемым: меняем state → вызываем render.

## Components

### 11. Что такое компонент?

Самостоятельная часть интерфейса со своими данными, DOM, событиями и поведением.

### 12. Что такое class?

Шаблон для создания объектов.

### 13. Что такое constructor?

Метод, который вызывается при создании экземпляра через `new`.

### 14. Что такое `this`?

Ссылка на конкретный экземпляр объекта или класса.

### 15. Почему ProductCard не должен напрямую менять Cart?

Чтобы не создавать сильную связанность компонентов.

### 16. Что такое CustomEvent?

Пользовательское событие для связи частей интерфейса.

### 17. Что хранится в `event.detail`?

Данные, переданные вместе с CustomEvent.

## Async

### 18. Что такое Promise?

Объект будущего результата асинхронной операции.

### 19. Какие состояния есть у Promise?

`pending`, `fulfilled`, `rejected`.

### 20. Что делает `then`?

Обрабатывает успешный результат.

### 21. Что делает `catch`?

Обрабатывает ошибку.

### 22. Что делает `finally`?

Выполняется в любом случае.

### 23. Что такое async/await?

Синтаксис для удобной работы с Promise.

### 24. Зачем нужен try/catch в async function?

Чтобы обработать ошибки await.

## Fetch and JSON

### 25. Что делает fetch?

Делает сетевой запрос и возвращает Promise.

### 26. Зачем проверять `response.ok`?

HTTP-ошибки могут прийти как обычный response.

### 27. Что делает `response.json()`?

Преобразует JSON-ответ в JavaScript-данные.

### 28. Чем JSON отличается от JavaScript object?

JSON — текстовый формат обмена данными. Object — структура данных в JavaScript.

## Forms

### 29. Почему submit слушают на form?

Submit срабатывает и по кнопке, и по Enter.

### 30. Что делает `preventDefault`?

Отменяет стандартное поведение браузера.

### 31. Что такое FormData?

Объект для чтения данных формы.

### 32. Почему input нужен name?

Без `name` поле не попадёт в FormData.

### 33. Зачем нужен trim?

Чтобы убрать пробелы в начале и конце строки.

### 34. Что такое validation?

Проверка данных перед отправкой.

### 35. Зачем regexp?

Для проверки строки по шаблону.

## UX States

### 36. Что такое loading state?

Состояние, когда данные загружаются.

### 37. Что такое error state?

Состояние ошибки.

### 38. Что такое success state?

Состояние успешного завершения действия.

### 39. Почему submit button нужно блокировать?

Чтобы пользователь не отправил форму несколько раз.

### 40. Почему форму не очищают при ошибке submit?

Пользователь потеряет введённые данные.

## Project Architecture

### 41. Как устроен Bangkok Express?

Данные загружаются, сохраняются в state, интерфейс рендерится из state, события меняют state и вызывают render.

### 42. Как перенести проект на TypeScript?

Добавить типы Product, CartItem, Order, OrderFormData, ApiResponse и типизировать функции.

### 43. Как перенести проект на React?

Разбить на компоненты, state перенести в useState, события передавать через props callbacks, fetch вынести в effects или data layer.
