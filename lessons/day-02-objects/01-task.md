# Day 02 — Объекты

## Ориентир по программе jsbasic

Тема из программы:

> Объекты

Объекты являются основной структурой данных в JavaScript, поэтому изучаем их подробно.

В этом уроке разбираем:

```text
- запись и чтение свойств
- перебор свойств
- передача объектов по ссылке
- копирование объектов
- методы объектов
- использование this
- деструктуризация объектов и массивов
```

## Тема дня

```text
объект → свойства → методы → this → копирование → деструктуризация
```

## Цель урока

После урока ты должен уверенно понимать:

- что такое объект;
- зачем объекты нужны во frontend;
- как читать свойства объекта;
- как записывать свойства объекта;
- как удалять свойства объекта;
- как перебирать свойства объекта;
- чем объект отличается от примитивного значения;
- что значит “объект передаётся по ссылке”;
- как сделать поверхностную копию объекта;
- что такое метод объекта;
- как работает `this` внутри метода;
- как работает деструктуризация объектов и массивов.

## Что делаем

Создаём модель данных для будущего проекта **Bangkok Express**.

Практика:

```text
Bangkok Express Product / Cart / Order Objects
```

Ты будешь работать с объектами:

```text
product
cart
customer
order
```

## Что должен уметь код

В `script.js` нужно самостоятельно описать логику:

```text
1. Создать объект product
2. Вывести данные product на страницу
3. Изменить цену товара
4. Добавить новое свойство товара
5. Удалить лишнее свойство
6. Перебрать свойства товара
7. Создать копию товара
8. Показать разницу между ссылкой и копией
9. Создать объект cart с методами
10. Использовать this внутри методов cart
11. Создать объект order
12. Использовать деструктуризацию
13. Вывести summary заказа на страницу
```

## Какие знания закрепляем

```text
- object literal
- key / value
- чтение через точку
- чтение через квадратные скобки
- запись свойств
- удаление свойств через delete
- перебор через for...in
- Object.keys()
- Object.values()
- Object.entries()
- передача объекта по ссылке
- копирование через spread
- методы объекта
- this
- деструктуризация объекта
- деструктуризация массива
- textContent
- console.log для проверки объектов
```

## Почему это важно для frontend

Во frontend почти все данные удобно хранить объектами:

```text
product
user
cart
order
settings
formData
apiResponse
```

В Bangkok Express объектами будут:

```text
- блюдо
- категория
- корзина
- позиция корзины
- покупатель
- заказ
```

Если ты плохо понимаешь объекты, дальше будет сложно с массивами товаров, корзиной, API, TypeScript, React props и state.

## Критерий готовности

Задача выполнена, если:

- объект `product` создан самостоятельно;
- свойства читаются через точку и квадратные скобки;
- новое свойство добавляется;
- существующее свойство изменяется;
- лишнее свойство удаляется;
- свойства объекта перебираются;
- есть пример передачи объекта по ссылке;
- есть пример копирования объекта;
- объект `cart` содержит методы;
- методы `cart` используют `this`;
- деструктуризация используется минимум 2 раза;
- данные выводятся на страницу;
- в консоли нет неожиданных ошибок;
- студент может объяснить каждую строку своего JavaScript-кода.
