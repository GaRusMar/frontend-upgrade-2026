# Day 30 Questions: final task manager

## Вопросы для самопроверки
1. Какие данные здесь являются состоянием?
2. Какие события управляют состоянием?
3. Как обновляется интерфейс после изменения состояния?
4. Где можно уменьшить дублирование кода?

## Ответы
1. Состояние — это данные, от которых зависит UI задачи.
2. События определяются интеракциями пользователя (клик, ввод и т.д.).
3. Через отдельную функцию обновления интерфейса.
4. Вынести повторяемые части в функции.

## Перед коммитом я должен объяснить
- Что хранится в состоянии.
- Как работает каждый обработчик.
- Как UI синхронизируется с состоянием.
