## Почему это важно для frontend

Большие интерфейсы нельзя удобно писать одним огромным файлом.

Их нужно делить на независимые части:

```text
ProductCard
Cart
Modal
Carousel
RibbonMenu
OrderForm
```

В Bangkok Express компонентами будут:

```text
- карточка товара
- корзина
- карусель
- ленточное меню
- модальное окно
- форма заказа
```

Это подготовка к React: React тоже строится на компонентах. Если ты понимаешь компоненты без React, потом легче понять props, state, events, render и composition.

## Критерий готовности

Задача выполнена, если:

- есть класс `ProductCard`;
- есть класс `Cart`;
- есть класс `Carousel`;
- каждый класс отвечает только за свою часть интерфейса;
- `ProductCard` создаёт DOM карточки товара;
- `Cart` хранит состояние корзины;
- `Carousel` хранит текущий слайд;
- `ProductCard` генерирует `CustomEvent`;
- `Cart` обновляется после события `product-add`;
- `event.detail.product` используется для передачи товара;
- `bubbles: true` используется осознанно;
- `render` вызывается после изменения состояния;
- код не превращён в один большой обработчик;
- в консоли нет неожиданных ошибок;
- студент может объяснить `class`, `constructor`, `new`, `this`, `render`, `CustomEvent`, `modules`.
