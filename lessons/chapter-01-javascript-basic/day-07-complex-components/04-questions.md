## Ответы

1. Компонент со своим состоянием, DOM, событиями, поведением и edge cases.
2. У простой кнопки обычно один обработчик, у сложного компонента несколько состояний и сценариев.
3. Данные, которые описывают текущее состояние компонента.
4. У modal есть open state, overlay, content, close button и несколько способов закрытия.
5. Кнопка close, overlay click, Escape.
6. Через `event.target === event.currentTarget`.
7. Пользователь может кликать по содержимому modal, и это не должно закрывать окно.
8. Слушать `keydown` и проверять `event.key === 'Escape'` и `isModalOpen`.
9. Горизонтальное меню категорий.
10. Текущую выбранную категорию.
11. Визуальное состояние выбранного элемента.
12. Компонент выбора значения из диапазона.
13. `input` срабатывает сразу при изменении значения.
14. События указателя: мышь, touch, стилус в одном API.
15. Один API покрывает разные устройства ввода.
16. Подсказки во время ввода текста.
17. Чтобы убрать пробелы и сделать поиск нечувствительным к регистру.
18. Иначе пользователь увидит нерелевантный полный список.
19. Состояние, когда query есть, но совпадений нет.
20. Индекс активной подсказки.
21. Выбрать активную подсказку.
22. Закрыть/очистить suggestions.
23. Откладывает выполнение функции до паузы в вызовах.
24. Чтобы не запускать поиск и render на каждую букву мгновенно.
25. Ограничивает частоту выполнения функции.
26. Scroll происходит очень часто и может перегрузить интерфейс.
27. Debounce ждёт паузу, throttle выполняет не чаще заданного интервала.
28. `dragstart`, `dragover`, `dragleave`, `drop`.
29. Чтобы разрешить drop.
30. Событие меняет состояние, потом интерфейс перерисовывается.
