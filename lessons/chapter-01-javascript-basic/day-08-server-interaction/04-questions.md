## Ответы

1. Собирает пользовательские данные и отправляет их.
2. Событие отправки формы.
3. Submit срабатывает и по кнопке, и по Enter.
4. Отменяет стандартную отправку и перезагрузку страницы.
5. Коллекция полей формы.
6. Объект для чтения данных формы.
7. Без name поле не попадёт в FormData.
8. Убирает пробелы в начале и конце строки.
9. Потому что пользователь фактически ничего не ввёл.
10. Проверка данных перед отправкой.
11. Frontend-валидацию можно обойти, сервер отвечает за безопасность.
12. Объект, где ключ — поле, значение — текст ошибки.
13. `Object.keys(errors).length > 0`.
14. Шаблон для проверки строки.
15. `pattern.test(value)`.
16. Метод для сетевого запроса.
17. Promise.
18. Выполняется при успешном завершении Promise.
19. Выполняется при ошибке Promise.
20. Выполняется после then/catch в любом случае.
21. HTTP-ошибки могут прийти как обычный response.
22. Превращает JSON-ответ в JS-данные.
23. Текстовый формат обмена данными.
24. JSON — текстовый формат, JS object — структура данных в JavaScript.
25. GET получает данные, POST отправляет данные.
26. Сообщает серверу формат тела запроса.
27. Превращает JS object в JSON-строку.
28. Состояние загрузки.
29. Состояние ошибки.
30. Состояние успешного завершения.
31. Состояние отправки формы.
