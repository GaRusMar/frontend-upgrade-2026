# Day 04 — DOM-модель

## Ориентир по программе jsbasic

Тема из программы:

> DOM-модель

Переходим к работе со страницей. Дальше изучаем возможности JavaScript в процессе построения интерфейсов в браузере.

На этом занятии изучаем DOM-модель документа и методы работы с ней. Они нужны, чтобы отображать кнопки, меню и другие компоненты, а также динамически их менять.

В этом уроке разбираем:

```text
- DOM-модель документа
- методы работы с DOM
- создание элементов
- вставка элементов на страницу
- изменение элементов на странице
- атрибуты
- свойства
- управление классами
- простые задачи по генерации компонентов интерфейса
- DOM и инструменты разработчика в браузере
- динамическая генерация элементов интерфейса
```

## Тема дня

```text
DOM → поиск элементов → создание элементов → атрибуты → classList → динамический интерфейс
```

## Цель урока

После урока ты должен уверенно понимать:

- что такое DOM;
- как браузер превращает HTML в дерево объектов;
- как найти элемент через `querySelector`;
- как найти несколько элементов через `querySelectorAll`;
- чем `textContent` отличается от `innerHTML`;
- как создать элемент через `createElement`;
- как добавить элемент на страницу через `append`;
- как менять атрибуты;
- как работать с `dataset`;
- как менять CSS-классы через `classList`;
- как сгенерировать интерфейс из массива данных;
- как проверять DOM в DevTools.

## Что делаем

Создаём динамическую отрисовку меню для будущего проекта **Bangkok Express**.

Практика:

```text
Bangkok Express Product Card Renderer
```

Ты будешь работать с:

```text
products[]
categories[]
product cards
category badges
DOM nodes
```

## Что должен уметь код

В `script.js` нужно самостоятельно описать логику:

```text
1. Создать массив products
2. Создать массив categories
3. Найти контейнеры на странице
4. Очистить контейнеры перед render
5. Создать карточку товара через createElement
6. Заполнить карточку данными товара
7. Добавить атрибуты и dataset
8. Добавить CSS-классы через classList
9. Создать список категорий
10. Отрисовать все карточки товаров
11. Отрисовать пустое состояние, если товаров нет
12. Отрисовать featured product
13. Проверить DOM через DevTools
```

## Почему это важно для frontend

Frontend — это не только данные. Frontend должен показывать данные пользователю.

В Bangkok Express DOM понадобится для:

```text
- генерации карточек товаров
- генерации меню категорий
- вывода цены
- вывода признаков spicy / nuts / vegetarian
- показа пустого состояния
- обновления интерфейса при фильтрах
- подготовки к компонентной архитектуре
```

Позже во фреймворках, например React, ты не будешь вручную писать `createElement` так часто. Но понимание DOM нужно, чтобы понимать, что реально происходит в браузере.

## Критерий готовности

Задача выполнена, если:

- контейнеры находятся через `querySelector`;
- карточки товаров создаются через `createElement`;
- данные товара выводятся через `textContent`;
- карточки получают CSS-классы через `classList`;
- карточки получают `data-product-id`;
- категории генерируются динамически;
- список товаров очищается перед новым render;
- пустое состояние отображается, если товаров нет;
- featured product отображается отдельно;
- DOM можно проверить во вкладке Elements;
- в консоли нет неожиданных ошибок;
- студент может объяснить каждую строку своего JavaScript-кода.
