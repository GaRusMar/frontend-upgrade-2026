# Day 02 — Codex

## Что можно отдать Codex

Codex можно попросить:

- создать структуру папки урока;
- создать `index.html`;
- создать `style.css`;
- создать пустой `script.js` с TODO;
- проверить мой код после самостоятельной реализации;
- объяснить ошибку из консоли;
- объяснить, почему объект изменился по ссылке;
- проверить, правильно ли используется `this`;
- предложить рефакторинг после того, как я сам написал решение.

## Что студент пишет сам

Самостоятельно пишу:

- объект `product`;
- чтение свойств через точку;
- чтение свойств через квадратные скобки;
- запись свойств;
- удаление свойств;
- перебор свойств;
- `Object.keys`;
- `Object.values`;
- `Object.entries`;
- пример передачи объекта по ссылке;
- пример копирования объекта;
- объект `cart`;
- методы объекта `cart`;
- использование `this`;
- объект `order`;
- деструктуризацию объекта;
- деструктуризацию массива;
- вывод данных на страницу.

## Что нельзя отдавать Codex

Codex нельзя просить:

- полностью написать `script.js`;
- написать все объекты за меня;
- написать методы cart за меня;
- исправить код без объяснения причины;
- заменить моё понимание готовым решением;
- коммитить код, который я не могу объяснить.

## Workflow

```text
1. Прочитать 01-task.md

8. Только после этого просить Codex сделать review
9. Исправить ошибки
10. Сделать коммит
```
