## Ответы

1. Компонент — самостоятельная часть интерфейса.
2. Чтобы делить проект на понятные части, переиспользовать код и проще расширять приложение.
3. Render-функция обычно только рисует DOM, а компонент хранит данные, DOM, методы и события.
4. Класс — шаблон для создания объектов.
5. `constructor` выполняется при создании экземпляра и задаёт начальные данные.
6. `new` создаёт новый экземпляр класса.
7. Экземпляр — конкретный объект, созданный по классу.
8. Метод класса — функция внутри класса.
9. `this` — ссылка на конкретный экземпляр класса.
10. Чтобы хранить корневой DOM-элемент компонента.
11. Создаёт или обновляет DOM компонента.
12. Потому что метод имеет доступ к данным компонента через `this`.
13. ProductCard — компонент карточки товара.
14. За DOM карточки, вывод товара и событие Add to cart.
15. Чтобы не создавать сильную связанность между компонентами.
16. Это когда компоненты общаются через события или внешний код, но не зависят от внутреннего устройства друг друга.
17. Cart — компонент корзины.
18. Items корзины и контейнер для вывода.
19. Пройти по items и сложить `price * quantity`.
20. Carousel — компонент карусели.
21. Slides, currentIndex и контейнер.
22. При выходе за конец возвращается к 0, при уходе ниже 0 переходит к последнему индексу.
23. Пользовательское событие.
24. Данные, которые передаются вместе с событием.
25. Отправляет событие из DOM-элемента.
26. Чтобы событие всплывало к родительским элементам.
27. Через `addEventListener('event-name', handler)`.
28. Cart не узнает, какой товар нужно добавить.
29. Родительский контейнер может не поймать событие.
30. Отдельный JS-файл, который может экспортировать и импортировать код.
31. Позволяет отдать функцию, класс или данные из файла.
32. Позволяет получить код из другого файла.
33. Чтобы браузер воспринимал script как модуль.
34. `components/ProductCard`, `components/Cart`, `components/Carousel`, `data`, `utils`, `index.js`.
35. React тоже строится на компонентах, которые получают данные, рендерят интерфейс и реагируют на события.
