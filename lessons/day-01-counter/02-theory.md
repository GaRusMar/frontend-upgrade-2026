# Day 01 Theory: Counter

## const
const используем для переменных, которые не должны переназначаться (например, ссылки на DOM-элементы).

## let
let используем для данных, которые меняются. В этом уроке это count.

## querySelector
querySelector ищет первый элемент по CSS-селектору. Если элемент не найден, вернёт null.

## addEventListener
addEventListener связывает событие и функцию-обработчик. Для кнопок счётчика используется событие click.

## count как состояние
count хранит текущее значение счётчика. Это состояние интерфейса.

Правило: изменили count -> обновили UI.

## textContent
textContent меняет текст элемента. Через него показываем текущее значение count в интерфейсе.

## classList
classList добавляет и удаляет CSS-классы. Это нужно, чтобы менять цвет значения для positive и negative.

## updateCounter
updateCounter — функция синхронизации состояния и UI.

Обычно она:
- обновляет textContent;
- переключает классы по знаку числа.

## Частые ошибки
1. Объявить count через const и потом пытаться менять.
2. Изменить count, но забыть обновить textContent.
3. Неверный селектор в querySelector.
4. Повесить обработчик на null.
5. Дублировать UI-обновление в каждом обработчике вместо updateCounter.
6. Добавлять классы, но не снимать противоположные.
