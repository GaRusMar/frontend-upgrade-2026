## Что делаем

Создаём простой счётчик:

```text
[-1] [0] [+1] [Reset]
```

Поведение:

- кнопка `+1` увеличивает значение;
- кнопка `-1` уменьшает значение;
- кнопка `Reset` сбрасывает значение в `0`;
- если значение больше `0` — число зелёное;
- если значение меньше `0` — число красное;
- если значение равно `0` — обычный цвет.

## Теория простыми словами

### const

`const` используется, когда переменную не нужно переопределять.

Пример:

```js
const plusButton = document.querySelector('.counter__button--plus')
```

Важно: `const` запрещает переопределить переменную, но не делает DOM-элемент замороженным.

---

### let

`let` используется для значений, которые будут меняться.

```js
let count = 0
```

В счётчике число меняется после каждого клика, поэтому нужен `let`.

---

### querySelector

`querySelector` ищет элемент на странице по CSS-селектору.

```js
const counterValue = document.querySelector('.counter__value')
```

Если элемент найден — вернётся DOM-элемент.

Если элемент не найден — вернётся `null`.

---

### addEventListener

`addEventListener` позволяет выполнить код после действия пользователя.

```js
plusButton.addEventListener('click', () => {
  // код выполнится после клика
})
```

Функция внутри не запускается сразу. Она ждёт действие пользователя.

---

### count как состояние

`count` — это состояние счётчика.

```js
let count = 0
```

Состояние — это данные, от которых зависит интерфейс.

Если `count` меняется, интерфейс тоже нужно обновить.

---

### textContent

`textContent` меняет текст внутри элемента.

```js
counterValue.textContent = count
```

---

### classList

`classList` управляет CSS-классами элемента.

```js
counterValue.classList.add('counter__value--positive')
counterValue.classList.remove('counter__value--negative')
```

---

### Функция updateCounter

После каждого действия нужно обновлять:

- текст счётчика;
- цвет счётчика.

Чтобы не дублировать код, создаём функцию:

```js
function updateCounter() {
  // обновление текста и классов
}
```

## Частые ошибки

### 1. Забыть точку в селекторе

Неправильно:

```js
document.querySelector('counter__value')
```

Правильно:

```js
document.querySelector('.counter__value')
```

---

### 2. Изменить count, но не обновить DOM

Неполный вариант:

```js
count = count + 1
```

Нужно:

```js
counterValue.textContent = count
```

---

### 3. Перепутать = и ===

`=` — присваивание.

`===` — сравнение.

---

### 4. Оставить два цветовых класса одновременно

Ошибка:

```text
counter__value--positive
counter__value--negative
```

Нужно следить, чтобы активным был только один класс или ни одного.

---

### 5. Повесить обработчик на null

Если элемент не найден, переменная будет равна `null`.

Тогда код сломается:

```js
plusButton.addEventListener('click', () => {})
```

1. Почему для кнопок лучше использовать `const`?
2. Почему для `count` нужен `let`?
3. Что делает `querySelector`?
4. Что вернёт `querySelector`, если элемент не найден?
5. Когда выполняется функция внутри `addEventListener`?
6. Чем `textContent` отличается от `innerHTML`?
7. Зачем нужен `classList`?
8. Почему изменение переменной не меняет страницу автоматически?
9. Зачем нужна функция `updateCounter`?
10. Как убрать конфликт между positive и negative классами?

---

## Ответы для самопроверки

1. Потому что ссылки на DOM-элементы не должны переопределяться.
2. Потому что значение счётчика меняется.
3. Ищет элемент на странице по CSS-селектору.
4. `null`.
5. После действия пользователя, например после клика.
6. `textContent` меняет текст, `innerHTML` вставляет HTML-разметку.
7. Чтобы добавлять и удалять CSS-классы.
8. Потому что JS-переменная и DOM — разные вещи. DOM нужно обновлять отдельно.
9. Чтобы не дублировать обновление текста и цвета.
10. Перед добавлением одного класса нужно удалить противоположный.

## Критерий готовности

- кнопка `+1` увеличивает значение;
- кнопка `-1` уменьшает значение;

- при `0` оба цветовых класса удалены;
- повторяющаяся логика вынесена в функцию;
- в консоли браузера нет ошибок.
